# Storybook (스토리북 생성 모듈)

## 역할

서버에 저장된 Q&A 데이터를 모아 부모님의 인생 스토리북으로 생성합니다.

## 핵심 기능

1. **데이터 수집**: 서버에서 특정 부모님의 Q&A 데이터 조회
2. **스토리 구성**: Q&A를 주제별/시간순으로 정렬하여 스토리 구성
3. **템플릿 렌더링**: Jinja2 템플릿으로 HTML 스토리북 생성
4. **PDF 변환**: HTML을 PDF로 변환 (인쇄/공유용)
5. **이미지 삽입**: 가족 사진 또는 AI 생성 삽화 추가

## 구성해야 할 파일

| 파일/폴더 | 설명 |
|------|------|
| `generator.py` | 스토리북 생성 핵심 로직 |
| `templates/` | Jinja2 HTML 템플릿 |
| `exporter.py` | PDF/EPUB 변환 로직 |
| `image_generator.py` | AI 이미지 생성 (Phase 2) |
| `requirements.txt` | Python 의존성 |

## 스토리북 구조 (제안)

```
표지: "[부모님 이름]의 이야기"
  ├── 1장: 유년기의 기억
  │     ├── Q: "어린 시절 가장 행복했던 기억은?"
  │     ├── A: (부모님 응답)
  │     └── [삽화 또는 사진]
  ├── 2장: 청춘의 꿈
  │     ├── Q: "스무 살 적 가장 큰 꿈은?"
  │     ├── A: (부모님 응답)
  │     └── [삽화 또는 사진]
  ├── 3장: 사랑과 가족
  │     └── ...
  ├── 4장: 인생의 지혜
  │     └── ...
  └── 맺음말: 가족이 전하는 말
```

## 기술 스택

- **템플릿 엔진**: Jinja2
- **PDF 변환**: WeasyPrint 또는 Playwright (Chrome 렌더링)
- **이미지 생성**: DALL-E API / Stable Diffusion (Phase 2)

## 다른 모듈과의 관계

- **server**: 서버 API에서 Q&A 데이터 조회
- **child-app**: 생성된 스토리북을 자식 앱에서 열람
- **tts**: 스토리북을 오디오북으로 변환 시 TTS 활용 (Phase 2)
- **shared**: 스토리북 데이터 스키마 공유

## 개발 시 구현해야 할 내용

- [ ] 서버에서 Q&A 데이터 조회 함수
- [ ] Q&A를 챕터/페이지로 구성하는 로직
- [ ] Jinja2 HTML 템플릿 작성 (표지, 챕터, 페이지, 맺음말)
- [ ] HTML → PDF 변환 (WeasyPrint)
- [ ] 사진 업로드 및 삽입 기능
- [ ] 스토리북 메타데이터 (제목, 부제, 저자, 생성일)
- [ ] (Phase 2) AI 이미지 생성으로 삽화 자동 생성
- [ ] (Phase 2) 오디오북 생성 (TTS + 배경음악)

## 확장 아이디어

- 다양한 디자인 테마 (감성, 모던, 클래식, 수채화)
- 가족 레시피북: 음식 관련 Q&A만 모아 별도 레시피북 생성
- 3세대 스토리북: 조부모 → 부모 → 자식으로 이어지는 연결형 스토리북
- 인쇄 주문 연동: 완성된 스토리북을 실물 책으로 인쇄 주문
- EPUB 포맷 지원: 전자책 리더기에서 읽을 수 있도록
