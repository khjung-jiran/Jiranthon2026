## 역할

부모님과 자식이 모두 사용하는 통합 앱입니다. 로그인 시 역할에 따라 UI가 분기됩니다.

## 역할별 모드

### 부모 모드 (Parent Mode)

- **질문 수신**: 자식이 보낸 질문을 TTS로 듣거나 텍스트로 확인
- **응답 입력**: 음성(STT) 또는 텍스트로 답변
- **고령자 친화적 UI**: 큰 글씨, 단순 네비게이션, 음성 안내

### 자식 모드 (Child Mode)

- **질문 작성**: 직접 작성 또는 자동 생성 질문 선택
- **응답 확인**: 부모님 응답을 텍스트/음성으로 확인
- **스토리북 열람**: 누적된 Q&A를 스토리북 형태로 열람
- **질문 관리**: 전송한 질문 목록, 답변 대기/완료 상태

## 기술 스택 (제안)

- **Framework**: React + TypeScript
- **Styling**: TailwindCSS
- **상태관리**: Zustand
- **API 통신**: Axios / TanStack Query
- **빌드**: Vite

## 역할 분기 구조 (제안)

```
로그인 → 역할 선택 (또는 계정 정보에 따라 자동)
  ├── parent → 부모 모드 (QuestionList → QuestionDetail → Respond)
  │            └── Calendar (사진 확인, 음성/텍스트 댓글)
  └── child  → 자식 모드 (Dashboard → ComposeQuestion / ResponseList / Storybook)
               └── Calendar (사진 업로드, 음성/텍스트 댓글)
```

## 다른 모듈과의 관계

- **server**: 질문/응답/스토리북/캘린더 API 호출
- **stt**: 부모 모드 음성 응답 및 캘린더 음성 댓글 시 STT 모듈 호출
- **tts**: 부모 모드에서 질문 수신 시 TTS로 발화
- **question-engine**: 자식 모드에서 질문 생성 API 호출
- **storybook**: 자식 모드에서 스토리북 열람
- **calendar**: 부모/자식 모두 캘린더 탭에서 사진 확인 및 댓글 작성
- **shared**: 공통 타입, API 스키마 참조

## UI/UX 원칙

### 부모 모드 (고령자 친화적)

- 기본 폰트 사이즈 20px 이상
- 최대 2단계 네비게이션 깊이
- 고대비 색상 조합
- 모든 주요 액션에 음성 피드백
- 터치 버튼 최소 60px
- 진행 상태 표시 ("질문을 듣고 계세요...", "답변을 말씀하세요...")

### 자식 모드

- 모던하고 깔끔한 디자인
- 대시보드 형태의 정보 요약
- 빠른 질문 작성 플로우

## 확장 아이디어

- 한 기기에서 역할 전환 (방문 시 자식이 부모 모드로 대필)
